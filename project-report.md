# Отчет по проекту: Корпоративный LLM-ассистент (RAG)

## Резюме

Три документа описывают архитектуру маркетингового сайта для B2B-продажи услуги внедрения корпоративного LLM-ассистента в закрытом контуре. Документы хорошо структурированы и содержат согласованную методологию "Council + Regularization + Verification".

---

## Сильные стороны проекта

### 1. Чёткое позиционирование
- **Главный дифференциатор:** "Ответы со ссылками на источники" — отстройка от галлюцинаций ChatGPT
- **Безопасность:** on-prem / облако РФ / закрытый контур
- **Прозрачность:** явный отказ от непроверяемых обещаний (проценты ROI без кейсов)

### 2. Продуманная архитектура
- MVP-подход: 3 страницы (Главная + Кейсы + Запрос демо)
- Варианты масштабирования (A/B/C) с чёткими критериями выбора
- Правильное решение: отказ от публичного интерактивного демо (AppSec риски)

### 3. Операционная зрелость
- SRE checklist (отказоустойчивость формы, мониторинг)
- AppSec checklist (threat model, least privilege)
- Verification layer ([НЕПОДТВЕРЖДЕНО] маркеры)

---

## Выявленные разрывы и рекомендации

| Область | Проблема | Рекомендация |
|---------|----------|--------------|
| **Дизайн-система** | Документы требуют "не придумывать стили", но система не предоставлена | Создать полноценную дизайн-систему (см. приложение) |
| **Контент** | Кейсы, логотипы, сроки помечены [НЕПОДТВЕРЖДЕНО] | Запустить контент-спринт параллельно с разработкой |
| **Интеграции** | Email/CRM не определены | Выбрать: Resend + Supabase (минимальный стек) |
| **Analytics** | Упомянуты события, но нет event-схемы | Определить: `lead_submit`, `cta_click`, `scroll_depth` |

---

## Рекомендуемая последовательность запуска

```
Неделя 1-2: Дизайн-система + контент-каркас + форма
Неделя 3:   Главная страница (MVP)
Неделя 4:   Кейсы + интеграция формы
Неделя 5:   QA + Lighthouse + Security audit
Запуск:     Вариант A (MVP)
```
