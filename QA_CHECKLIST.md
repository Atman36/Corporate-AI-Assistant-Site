# Production QA Checklist

## Форма заявки
- Валидная заявка: корректные имя/компания/email/телефон, согласие отмечено, комментарий опционален.
- Невалидная заявка: пустые обязательные поля, неправильный email, короткий/битый телефон.
- Проверка анти-спам: honeypot поле не видно пользователю и блокирует бот-трафик.
- Проверка rate-limit: после серии быстрых отправок API возвращает `429 RATE_LIMITED`.

## Устойчивость интеграции
- Telegram недоступен или неверный токен: API возвращает `502 TELEGRAM_UNAVAILABLE`, UI показывает понятную ошибку.
- Telegram не настроен (нет env): API возвращает `503 TELEGRAM_NOT_CONFIGURED`.
- CSRF токен просрочен/некорректен: API возвращает `403 CSRF_MISMATCH`.

## Безопасность
- Неожиданные поля в payload: API отклоняет запрос (`400 UNEXPECTED_FIELDS`).
- Origin не совпадает с доменом: API отклоняет запрос (`403 FORBIDDEN_ORIGIN`).
- Проверить, что пользовательский комментарий не рендерится как HTML в UI.

## Mobile + Accessibility
- Проверить форму и CTA на ширинах `360`, `390`, `768`, `1024`, `1440`.
- Все поля доступны с клавиатуры, ошибки озвучиваются (aria-invalid + aria-describedby).
- Кнопка отправки имеет состояние loading и блокируется во время запроса.

## Smoke в проде
- Отправить реальную тестовую заявку.
- Убедиться, что сообщение приходит в Telegram c `Request ID`.
- Проверить логи: есть событие `lead accepted`, нет PII в логах.
