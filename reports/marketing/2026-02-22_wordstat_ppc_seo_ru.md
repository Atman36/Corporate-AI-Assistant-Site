Регион Wordstat: **Россия (225)**  
Источники данных: выгрузки Yandex Wordstat API (файлы `reports/wordstat/*.csv`) + ручная фильтрация по интенту.

## 1) Короткий вывод

- Спрос “прямыми словами” про **«закрытый контур»** как маркетинговый термин в поиске **сильно шумный**: большая часть показов — не про ИБ/AI (отопление, радиотехника и т.п.). Это плохая базовая семантика для поиска.  
- Коммерческий спрос лучше “ловится” через формулировки: **«внедрение ИИ/нейросетей»**, **«разработка чат‑бота с ИИ»**, **«локальная/ on‑prem LLM»**, **«enterprise search / системы поиска по документам»**, **«корпоративный ChatGPT»**, **«RAG внедрение»**.
- По запросам вида **«чатбот по документам»** спрос есть, но относительно небольшой — это скорее **точечные BOFU‑группы** + хорошая тема для SEO‑лендингов и кейсов.

## 3) Что выкинуть (нецелевые интенты) и минус‑слова

### Типовые нецелевые категории

- **Обучение/курсы/практикумы** по нейросетям (и особенно бренды курсов).  
- **Промпты/промты** (люди хотят шаблоны запросов, не внедрение).  
- **“Бесплатно/скачать/онлайн”** (часто B2C).  
- **Вакансии/работа** (кандидаты, не заказчики).  
- **Telegram‑боты “для продаж/инфобиза”** — если ваш фокус именно корпоративные ассистенты по документам, лучше отсечь.

### Базовый список минус‑слов (старт)

`курс`, `обучение`, `практикум`, `интенсив`, `вебинар`, `нетология`, `skillbox`, `eduson`, `книга`, `скачать`, `бесплатно`, `промпт`, `промты`, `шаблон`, `вакансия`, `работа`, `резюме`, `как сделать`, `своими руками`, `python`, `пример` (часть TOFU можно оставить в РСЯ/SEO, но убрать из BOFU‑поиска).

Для “закрытый контур” (если будете тестировать как гипотезу) дополнительно: `отопление`, `колебательный`, `теплый пол`, `котел`, `дыхательный`, `радио`, `электротехника`.

## 4) Как продвигаться в рекламе (Yandex Direct): структура и тактика

### 4.1. Структура аккаунта (рекомендация)

1) **Поиск / BOFU — Внедрение ИИ**  
Ключи: “внедрение ai”, “внедрение нейросети”, “внедрение искусственного интеллекта в бизнес/компании”, “внедрение ai ассистента”.  
Лендинг: “Внедрение корпоративного AI‑ассистента / пилот 2–3 недели”.

2) **Поиск / BOFU — Разработка AI‑чатбота**  
Ключи: “разработка чат бота с ии”, “разработка интеллектуального чат бота”, “заказать/стоимость/под ключ”, “для бизнеса”.  
Лендинг: “Чат‑бот для сотрудников / по документам / база знаний + интеграции”.

3) **Поиск / MOFU‑BOFU — RAG / Enterprise search / Поиск по документам**  
Ключи: “системы поиска по документам”, “интеллектуальный поиск по документам”, “enterprise search”, “rag внедрение”.  
Лендинг: “RAG‑поиск по внутренним документам с источниками + RBAC/аудит”.

4) **Поиск / MOFU — On‑prem / локальная LLM**  
Ключи: “локальные llm”, “локальное развертывание llm”, “on prem решения” (только с сильной минусацией).  
Лендинг: “Развёртывание LLM в закрытом контуре (on‑prem / облако РФ)”.

5) **Поиск / Конкуренты и бренды (осторожно)**  
Ключи: “корпоративный chatgpt”, “chatgpt для бизнеса”, “gigachat для бизнеса”, “yandexgpt …”, “naumen enterprise search”.  
Лендинг: “Корпоративный ассистент с источниками и контролем доступа” + блок сравнения (без агрессивных заявлений).

6) **РСЯ (контент) + ретаргетинг**  
Аудитории: посетители one‑pager, скролл 50%+, клики “Запросить демо”, просмотр блока “Безопасность”.  
Креативы: “проверяемые ответы с источниками”, “RBAC/ACL и аудит”, “пилот 2–3 недели”, “интеграции SSO/LDAP/ServiceDesk/порталы”.

### 4.2. Что критично для конверсии в B2B

- **Разные лендинги под интент**: “внедрение ИИ” ≠ “разработка чат‑бота” ≠ “on‑prem LLM” ≠ “enterprise search”.  
- **Лид‑магниты**: “one‑pager (PDF)”, “чек‑лист ИБ для LLM в контуре”, “пример отчёта PoC” — как альтернативная конверсия.  
- **Сквозная аналитика**: Метрика + цели на формы/клики/скачивание PDF, коллтрекинг (если телефон), выгрузка в CRM.

Вариант 1 (внедрение):
- Заголовок: “Корпоративный AI‑ассистент в вашем контуре”

Вариант 3 (on‑prem LLM):
- Заголовок: “LLM on‑prem: развёртывание и интеграции”
- Текст: “Развёртывание в закрытом контуре, контроль доступа, логи. Подключаем источники и корпоративные системы по API.”

## 5) Рекомендации по SEO: что делать в первую очередь

### 5.1. Структура посадочных страниц (обязательный минимум)

1) “Корпоративный AI‑ассистент в закрытом контуре” (основной коммерческий лендинг)  
2) “RAG‑поиск по внутренним документам” (коммерческий)  
3) “On‑prem / локальная LLM для компании” (коммерческий)  
4) “AI‑чатбот для сотрудников / база знаний” (коммерческий)  
5) Страницы сценариев: “для поддержки/ITSM”, “для HR”, “для юристов/договоров”

### 5.2. Контент‑кластеры (чтобы собирать TOFU→MOFU)

- “Внедрение ИИ в бизнес”: статья‑гайд, чек‑лист, типовые ошибки, как считать эффект (без обещаний, с допущениями).  
- “Безопасность и ИБ”: RBAC/ACL, аудит, требования к контурам, журналирование, управление источниками.  
- “RAG на практике”: как обеспечить проверяемость ответов, как бороться с галлюцинациями (качественные формулировки, ограничения).  
- “Интеграции”: SSO/LDAP, ServiceDesk, Confluence/SharePoint, хранилища документов (каждую интеграцию — отдельной страницей, если реально поддерживаете).

### 5.3. Техническое SEO (быстрые победы)

- Сниппеты: FAQ‑блоки на коммерческих страницах (FAQ schema), понятные title/description под кластеры.  
- Внутренняя перелинковка: “решения → сценарии → безопасность → кейсы/демо”.  
- Доказательность: скрин/видео “ответ + источник” (ваш самый сильный differentiator), раздел “как мы делаем пилот” с артефактами (план, критерии, отчёт).

## 6) Мой анализ и выводы (практические)

1) **Не пытайтесь покупать спрос “закрытый контур” как главный ключ**: слишком много нерелевантных значений термина. “Закрытый контур” лучше использовать как **сообщение в тексте/оффере**, а не как ядро семантики.  
2) Самые перспективные точки входа в BOFU: **“внедрение ИИ/нейросетей”** и **“разработка чат‑бота с ИИ”** — там реальный коммерческий интент.  
3) Для вашего позиционирования (“проверяемые ответы с источниками”) самый “свой” кластер — **RAG/поиск по документам/enterprise search**, но он требует правильных формулировок и страниц под B2B‑термины.  
4) В SEO быстрее всего вырастет трафик через **TOFU‑материалы про внедрение ИИ** + “безопасность/контур”, но лиды будут приходить через **коммерческие лендинги** и ретаргет.  
5) В рекламе ключевое — **развести кампании по интенту** и жёстко минусовать обучение/промпты/бесплатно, иначе бюджет уйдёт в B2C‑спрос.
